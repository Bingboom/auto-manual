# Presentation Materials

This directory stores presentation source materials for business rollout, internal sharing, and customer-facing introductions around the auto-manual workflow.

Current structure:

- `01_problem.md`: pain points in the current manual creation workflow
- `02_solution.md`: structured-content solution and core flow
- `03_capability.md`: current implemented capabilities
- `04_case.md`: business case / example page
- `05_next_step.md`: near-term plan and rollout direction

Recommended usage:

- keep one topic per file so slides can be assembled or reordered easily
- store screenshots or exported figures in a future `assets/` folder under this directory
- keep these files focused on presentation narrative, not detailed implementation notes
- when the external messaging changes, update this directory together with the related user-guide docs

Branching guidance:

- use a short-lived docs branch such as `codex/docs-presentation-rollout`
- merge back to `main` through a PR after the deck outline is stable
